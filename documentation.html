Documentation

Resources Information

  • Base URL: http://localhost:3000

    • We provide the following functionality:

    1. Retrieve the full dataset

    Description: get a list of all items stored in the database
    Endpoint: /products
    HTTP methods: GET (Safe - this is a read-only operation, it does not alter the values of the database on the server)
    Content-Type: application/json
    Request Parameters: NONE
    Example Request:
    GET /products HTTP/1.1
    Host: localhost:3000
    User-Agent: Mozilla/5.0 Chrome/48.0.2564
     Response Codes:

  • 200: OK -request success.
    • Example response: (when they are 2 example items in the database)
      [
      {
      "id":2,
      "brand":"Fairphone",
      "model":"FP3",
      "os":"Android",
      "image":"https://upload.wikimedia.org/wikipedia/commons/thumb/c/c4/Fairphone_3_modules_on_display.jpg/320px-Fairphone_3_modules_on_display.jpg",
      "screensize":5.65
      },
      {
      "id":3,
      "brand":"Fairphone",
      "model":"FP3",
      "os":"Android",
      "image":"https://upload.wikimedia.org/wikipedia/commons/thumb/c/c4/Fairphone_3_modules_on_display.jpg/320px-Fairphone_3_modules_on_display.jpg",
      "screensize":5.65
      }
      ]
    • Example response: (when there are no items in the database)
      This is an empty table
  • 500: Internal Server Error -internal database error which will occur in situations such as the database does not exist.

    • 2. Add new product item

    Description: add data in the database for a new product item and assign a new id to the new product item
    Endpoint: /create
    HTTP methods: POST
    Content-Type: application/json
    Request Parameters:

  • data (JSON): the new product data in json format.
    • Example value:
      {
      "brand":"Fairphone",
      "model":"FP3",
      "os":"Android",
      "image":"https://upload.wikimedia.org/wikipedia/commons/thumb/c/c4/Fairphone_3_modules_on_display.jpg/320px-Fairphone_3_modules_on_display.jpg",
      "screensize":5.65
      }
    • Example Request:
    POST /create HTTP/1.1
    Host: localhost:3000
    User-Agent: Mozilla/5.0 Chrome/48.0.2564

    {"brand":"Fairphone","model":"FP3","os":"Android","image":"https://upload.wikimedia.org/wikipedia/commons/thumb/c/c4/Fairphone_3_modules_on_display.jpg320px-Fairphone_3_modules_on_display.jpg","screensize":5.65}     Response Codes:
  • 201: Created -The request has been fulfilled and resulted in a new resource being created.
    • Example response:
      Created successfully
  • 400: Bad Request -the request cannot be fulfilled due to bad syntax.
    • Example response:
      Double check your JSON query if anything is empty or undefined
  • 500: Internal Server Error -internal database error which will occur in situations such as the database does not exist.

    • 3. List item by ID

    Description: to list the data of a specific item by ID in the database
    Endpoint: /:id
    HTTP methods: GET
    Content-Type: application/json
    Request Parameters:

  • ID (number): the corresponding ID of the specific data
    • Example Request: (id=2)
    GET /2 HTTP/1.1
    Host: localhost:3000
    User-Agent: Mozilla/5.0 Chrome/48.0.2564
     Response Codes:
  • 200: OK -request success.
    • Example response:
      {
      "id":2,
      "brand":"Fairphone",
      "model":"FP3",
      "os":"Android",
      "image":"https://upload.wikimedia.org/wikipedia/commons/thumb/c/c4/Fairphone_3_modules_on_display.jpg/320px-Fairphone_3_modules_on_display.jpg",
      "screensize":5.65
      }
  • 404: Not Found -the requested id does not exist in the database.
    • Example response:
      Cannot find the id
  • 500: Internal Server Error -internal database error which will occur in situations such as the database does not exist.

    • 4. Update specific item by ID

    Description: change data of a specific item in the database by ID
    Endpoint: /update
    HTTP methods: PUT (idempotent - multiple identical requests will only result in one outcome on the server)
    Content-Type: application/json
    Request Parameters:

  • data (JSON): the new product data in json format.
    • Example value:
      {
      "id": 2,
      "brand":"Iphone",
      "model":"X",
      "os":"IOS",
      "image":"https://www.google.com/url?sa=i&url=https%3A%2F%2Fwww.apple.com%2Fnl%2Fnewsroom%2F2017%2F09%2Fthe-future-is-here-iphone-x%2F&psig=AOvVaw0oGmnjP725tb1oVc2UfXtS&ust=1643314206614000&source=images&cd=vfe&ved=0CAsQjRxqFwoTCOj_5c6c0PUCFQAAAAAdAAAAABAM",
      "screensize":5.8
      }
    • Example Request:
    PUT /update HTTP/1.1
    Host: localhost:3000
    User-Agent: Mozilla/5.0 Chrome/48.0.2564

    {"id": 2,"brand":"Iphone","model":"X","os":"IOS","image":"https://www.google.com/url?sa=i&url=https%3A%2F%2Fwww.apple.com%2Fnl%2Fnewsroom%2F2017%2F09%2Fthe-future-is-here-iphone-x%2F&psig=AOvVaw0oGmnjP725tb1oVc2UfXtS&ust=1643314206614000&source=images&cd=vfe&ved=0CAsQjRxqFwoTCOj_5c6c0PUCFQAAAAAdAAAAABAM","screensize":5.8}     Response Codes:
  • 204: No Content -the server successfully processed the request, but is not returning any content.
  • 400: Bad Request -the request cannot be fulfilled due to bad syntax.
    • Example response:
      Double check your JSON query if anything is empty or undefined
  • 404: Not Found -the requested id does not exist in the database.
    • Example response:
      Cannot find the id
  • 500: Internal Server Error -internal database error which will occur in situations such as the database does not exist.

    • 5. Delete data of item by ID

    Description: remove data of a specific item in the database by ID
    Endpoint: /delete/:id
    HTTP methods: DELETE (idempotent - multiple identical requests will only result in one outcome on the server)
    Content-Type: application/json
    Request Parameters:

  • ID (number): the corresponding ID of the to-be-deleted data
    • Example Request: (id=4)
    DELETE /delete/4 HTTP/1.1
    Host: localhost:3000
    User-Agent: Mozilla/5.0 Chrome/48.0.2564
     Response Codes:
  • 204: No Content -the server successfully processed the request, but is not returning any content.
  • 404: Not Found -the requested id does not exist in the database.
    • Example response:
      Cannot find the id
  • 500: Internal Server Error -internal database error which will occur in situations such as the database does not exist.

    • 6. Reset

    Description: reset the database to an empty one
    Endpoint: /reset
    HTTP methods: DELETE
    Content-Type: application/json
    Request Parameters: NONE
    Example Request:
    DELETE /reset HTTP/1.1
    Host: localhost:3000
    User-Agent: Mozilla/5.0 Chrome/48.0.2564
     Response Codes:

  • 204: No Content -the server successfully processed the request, but is not returning any content.
  • 500: Internal Server Error -internal database error which will occur in situations such as the database does not exist.
